PonSu

PonSu は、申請の状態遷移と監査（audit trail）を中核にしたワークフローアプリケーションです。
デモでは、外部認証/DB/課金要素を排除し、フロントのみの静的ホスティングで操作フローを確認できるようにしています。

スクリーンショットは準備中（公開後に差し替え）

デモで体験できること

  • 申請一覧：ID / タイトル / ステータス / 更新日時を確認
  • 申請詳細：内容・履歴を確認し、操作へ到達
  • 操作：提出 / 承認 / 却下 / 差し戻し / 再提出
  • 監査ログ：操作が時系列で追える（デモはローカル状態で再現）
  • テーマ：ライト/ダーク/システム切替

設計上の見せどころ（デモの方針）

デモはフロントのみで完結

デモは静的ホスティングを前提にし、外部認証/DB/API呼び出しを行いません。 そのため公開時のリスク（秘密情報混入・課金・個人情報の取り扱い）を物理的に最小化できます。

本番相当は別モードに分離

本番相当（バックエンド接続）とデモ（フロントのみ）をモードで切り替え可能な構成にし、 デモのためにアプリ本体の方向性を歪めないようにしています。
