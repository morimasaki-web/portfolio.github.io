PonSu

PonSu は、申請業務の状態遷移と監査ログ管理を一体化したワークフローアプリケーションです。
申請の「作成→提出→承認/却下→差し戻し→再提出」といった複雑なフローを、状態マシンで厳密に制御し、各操作をイベントとして記録します。
監査ログ（audit trail）により「誰が・いつ・どの操作をしたか」を時系列で可視化し、業務の透明性とトレーサビリティを確保します。

主な機能:
・申請ごとの状態遷移管理（状態マシンによる厳密な制御）
・提出・承認・却下・差し戻し・再提出などの操作履歴を監査ログとして記録
・ユーザー種別や申請状態に応じた操作権限の自動判定
・申請一覧・詳細・履歴のリアルタイム表示
・ライト/ダーク/システムのテーマ切替、アクセシビリティ配慮
・フロントエンドは React + TypeScript、状態管理は Context/Reducer パターンを採用

用途・導入メリット:
PonSuは、申請業務の標準化・効率化・監査対応を求める現場に最適です。
状態遷移や履歴管理をアプリケーションレベルで保証することで、属人化や手作業によるミスを防ぎ、監査やトラブル時の証跡確認も容易になります。
また、デモモードを備えており、実運用前に操作感や業務フローを安全に体騯することも可能です。

デモで体騯できること

  • 申請一覧：ID / タイトル / ステータス / 更新日時を確認
  • 申請詳細：内容・履歴を確認し、操作へ到達
  • 操作：提出 / 承認 / 却下 / 差し戻し / 再提出
  • 監査ログ：操作が時系列で追える（デモはローカル状態で再現）
  • テーマ：ライト/ダーク/システム切替
